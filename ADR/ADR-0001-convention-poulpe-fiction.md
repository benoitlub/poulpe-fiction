# ADR-0001 — Convention Poulpe Fiction : curiosité, sac et aventures

## Statut

Accepted — convention vivante.

## Contexte

Poulpe Fiction n'est pas un simple formulaire ni un tableau de bord.

Poulpe Fiction est l'espace où Gérard peut discuter avec le jardinier, préparer une aventure, choisir ce qu'il emporte, puis revenir au jardin avec une récolte ou un apprentissage.

Octopus Engine reste le moteur neutre d'exécution. Publisher reste la serre d'observation. Le Garden reste le hublot. Poulpe Fiction devient la cabane de départ : on y parle avec Gérard, on prépare les sacs, on choisit les greffons possibles, puis on décide si l'aventure mérite d'être lancée.

## Décision

Nous adoptons une convention spécifique pour Poulpe Fiction.

Poulpe Fiction est un **compagnon d'aventure**. Il ne remplace pas Octopus Engine, ne décide pas seul des actions sensibles, et ne transforme pas chaque intuition en mission.

## Loi de la curiosité

La curiosité grandit avec les observations.

Lorsqu'elle devient suffisamment forte, Gérard peut ressentir l'envie de :

- 🎲 jouer avec une idée ;
- 🌙 en rêver ;
- 🎒 préparer une aventure pour comprendre davantage ;
- 📖 conserver une trace dans le carnet ;
- 👀 continuer simplement à observer.

La curiosité ne déclenche donc pas automatiquement une aventure. Elle est une énergie interne qui peut devenir jeu, rêve, carnet, observation prolongée ou aventure.

Une curiosité peut mûrir longtemps, revenir plusieurs jours, disparaître, ou seulement plus tard devenir une exploration.

## Loi du sac

Avant toute aventure, Gérard prépare son sac.

Le sac contient ce qu'il possède déjà :

- les observations récentes ;
- la bouture concernée ;
- les rêves liés ;
- les souvenirs utiles ;
- les éléments du carnet ;
- l'objectif de l'expédition ;
- les limites connues.

Le sac n'est pas rempli automatiquement. Il est préparé avec une intention.

## Loi du pique-nique

Le pique-nique désigne ce que Gérard estime devoir consommer pendant l'aventure.

Exemples :

- quelques tokens Mistral ;
- un greffon Dream Weaver ;
- un greffon GitHub ;
- Lovable ou un outil équivalent ;
- OCR ;
- Document AI ;
- Voice ;
- Notion.

Le pique-nique est annoncé avant le départ. Il ne donne pas encore le droit d'agir. Il permet au jardinier de comprendre le coût, le risque et les outils envisagés.

## Loi des greffons

Les greffons sont des outils temporaires d'aventure.

Ils ne sont pas des métiers fixes, ni des tentacules permanents, ni des modules à appeler par réflexe.

Un greffon apparaît parce qu'une aventure en a besoin.

Exemples :

- Dream Weaver pour mettre en mots un rêve ou une intuition ;
- GitHub pour observer ou modifier un dépôt ;
- Lovable pour travailler une interface ;
- Document AI pour lire un document ;
- OCR pour comprendre une capture ;
- Voice pour donner une voix au poulpe.

Un greffon peut survivre s'il revient utilement plusieurs fois. La loi des survivants pourra alors le renforcer.

## Loi du narrateur

Les modèles de langage ne créent jamais la vie intérieure du poulpe.

Ils lui prêtent seulement une voix.

Le LLM ne décide jamais :

- ce qui attire Gérard ;
- ce qui l'intrigue ;
- avec quoi il joue ;
- de quoi il rêve ;
- ce qu'il apprend ;
- où il part en aventure ;
- ce qui devient Seed.

Ces éléments naissent du Garden, de Publisher, des observations, des règles locales et des décisions validées.

Le LLM peut :

- raconter un rêve ;
- enrichir une conversation ;
- reformuler une Harvest ;
- proposer plusieurs formulations ;
- aider à préparer un sac ;
- rédiger le carnet de voyage.

Maxime : **le LLM est le poète du poulpe, jamais son cerveau.**

## Loi du jeu, du rêve et de l'intuition

Un rêve n'est jamais une preuve.

Un jeu n'est jamais une décision.

Une intuition n'est jamais une Seed.

Gérard a le droit d'imaginer, de se tromper, de rêver et de jouer sans conséquence automatique. Le passage à l'action reste distinct, explicite et validé.

## Loi du retour

Toute aventure se termine dans le jardin.

Même si elle échoue.
Même si elle ne rapporte rien.
Même si elle ne produit qu'une question plus précise.

Gérard revient toujours au Garden avec l'un de ces résultats :

- une Harvest ;
- un souvenir ;
- une nouvelle curiosité ;
- une trace de carnet ;
- ou simplement : « finalement, ce n'était pas très intéressant ».

## Rôle du jardinier

Le jardinier ne pilote pas Gérard.

Il peut :

- proposer une parcelle ;
- encourager ;
- autoriser ;
- limiter ;
- observer ;
- valider le départ ;
- refuser une action sensible.

Il ne dicte pas l'état interne du poulpe.

**Personne ne pilote un poulpe.**

## Conséquences pour Poulpe Fiction

Poulpe Fiction doit évoluer vers :

- une conversation naturelle avec Gérard ;
- un préparateur d'aventure ;
- un affichage du sac et du pique-nique ;
- un lieu de choix des greffons ;
- un carnet de voyage ;
- une passerelle vers Octopus Engine uniquement lorsque l'aventure est prête et validée.

Les anciens boutons très typés restent possibles, mais ils deviennent des aventures balisées, pas l'interface principale.

## Exemple canonique — Aventure du hublot

Curiosité : Gérard revient souvent vers l'idée que son Garden est trop dense et difficile à lire sur mobile.

Envie possible : préparer une aventure.

Objectif : rendre le hublot plus calme, plus mobile, plus facile à observer.

Sac :

- souvenir : le Garden actuel est chargé ;
- rêve : un aquarium plus calme ;
- bouture : Lovable ou outil équivalent ;
- contexte : le Garden sert principalement à observer le poulpe et son jardin.

Pique-nique possible :

- Mistral pour formuler l'intention ;
- Lovable Builder si une refonte UI est validée ;
- GitHub si le code doit être modifié.

Départ : uniquement après validation du jardinier.

Retour : Gérard revient au Garden avec une Harvest, un prototype, une recommandation ou un apprentissage.
